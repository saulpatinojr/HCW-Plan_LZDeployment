# HCW Build Standards Reference

**Purpose**: Quick reference for Terraform (AVM) and Bicep (Microsoft Learn) standards when implementing Phases 1-5.

## Terraform Modules (AVM Standard)

### File Structure Template

```
terraform/modules/example-resource/
├── main.tf                    # Primary resource definition
├── variables.tf               # Input parameters
├── outputs.tf                 # Return values
├── locals.tf                  # (optional) Local values
├── terraform.tf               # Version constraints
├── deprecated_variables.tf    # (optional) Deprecated inputs
├── deprecated_outputs.tf      # (optional) Deprecated returns
├── README.md                  # Generated by terraform-docs
└── .terraform-docs.yml        # terraform-docs configuration
```

### terraform.tf — Version Constraints

```hcl
terraform {
  required_version = "~> 1.6"
  
  required_providers {
    azapi = {
      source  = "Azure/azapi"
      version = "~> 2.0"
    }
    azurerm = {
      source  = "hashicorp/azurerm"
      version = "~> 4.0"
    }
  }
}
```

### variables.tf — Proper Structure

```hcl
# ✅ CORRECT: Required variables first (alphabetical), then optional

variable "environment" {
  type        = string
  description = "Environment name (dev, test, prod)"
}

variable "location" {
  type        = string
  description = "Azure region for resource deployment"
}

variable "resource_group_name" {
  type        = string
  description = "Name of the resource group"
}

# Optional variables below (alphabetical)

variable "allowed_ip_addresses" {
  type        = list(string)
  default     = []
  nullable    = false
  description = "List of IP addresses allowed to access the resource. Default: empty (allow all)"
}

variable "enable_monitoring" {
  type        = bool
  default     = false
  nullable    = false
  description = "Enable Application Insights monitoring for the resource. Default: false"
}

variable "resource_tags" {
  type = object({
    cost_center = string
    owner       = string
    team        = string
  })
  description = <<-EOT
  Tags to apply to all resources. Required fields:
  - cost_center: Cost center code for billing
  - owner: Owner email or team name
  - team: Team responsible for resource
  EOT
}

# ❌ WRONG: Don't do these

# variable "enabled"  # No "enabled" flag to toggle entire module
# variable "module_depends_on"  # No pseudo-dependency variables
# variable "extra_config"  # type = any is forbidden (use precise types)
```

### outputs.tf — Discrete Attributes Only

```hcl
# ✅ CORRECT: Output computed attributes, not entire resources

output "id" {
  description = "The unique identifier of the resource"
  value       = azurerm_resource.example.id
}

output "name" {
  description = "The name of the deployed resource"
  value       = azurerm_resource.example.name
}

output "fqdn" {
  description = "The fully qualified domain name"
  value       = azurerm_resource.example.properties.fqdn
}

# For sensitive data
output "connection_string" {
  description = "Database connection string (sensitive)"
  value       = azurerm_resource.example.connection_string
  sensitive   = true
}

# For resources deployed with for_each, output as map
output "subnet_ids" {
  description = "Map of subnet names to their resource IDs"
  value = {
    for key, subnet in azurerm_subnet.example : key => subnet.id
  }
}

# ❌ WRONG: Don't export entire resources
# output "resource" {
#   value = azurerm_resource.example  # Exposes all properties, includes sensitive data
# }

# ❌ WRONG: Don't output values already in inputs (except name)
# output "location" {
#   value = var.location  # Already provided by user, adds no value
# }
```

### main.tf — Resource Ordering & Patterns

```hcl
# ✅ CORRECT: Order by dependency, use snake_case

# 1. Resources with no dependencies first
resource "azurerm_resource_group" "platform" {
  name     = var.resource_group_name
  location = var.location
  tags     = var.resource_tags
}

# 2. Parent resources before child resources
resource "azurerm_virtual_network" "example" {
  name                = "${var.environment}-vnet"
  address_space       = ["10.0.0.0/16"]
  location            = azurerm_resource_group.platform.location
  resource_group_name = azurerm_resource_group.platform.name
  tags                = var.resource_tags
}

# 3. Child resources of parent
resource "azurerm_subnet" "workload" {
  for_each             = var.subnet_map
  name                 = "${var.environment}-subnet-${each.key}"
  resource_group_name  = azurerm_resource_group.platform.name
  virtual_network_name = azurerm_virtual_network.example.name
  address_prefixes     = each.value.address_space
}

# 4. Resources that reference outputs of previous
resource "azurerm_network_interface" "vm" {
  count                     = var.create_vm ? 1 : 0  # Use count for conditionals
  name                      = "${var.environment}-nic"
  location                  = azurerm_resource_group.platform.location
  resource_group_name       = azurerm_resource_group.platform.name
  ip_configuration {
    name                          = "primary"
    subnet_id                     = azurerm_subnet.workload["primary"].id
    private_ip_address_allocation = "Dynamic"
  }
  tags = var.resource_tags

  depends_on = [azurerm_virtual_network.example]  # Explicit if needed
}

# ✅ Dynamic blocks for conditional nested objects
resource "azurerm_container_app" "web" {
  name                         = "${var.environment}-app"
  location                     = azurerm_resource_group.platform.location
  resource_group_name          = azurerm_resource_group.platform.name
  container_app_environment_id = azurerm_container_app_environment.platform.id

  template {
    container {
      name   = "web"
      image  = var.container_image
      cpu    = "0.5"
      memory = "1Gi"

      # Dynamic block for optional features
      dynamic "env" {
        for_each = var.enable_monitoring ? [1] : []
        content {
          name  = "APPLICATIONINSIGHTS_CONNECTION_STRING"
          value = azurerm_application_insights.platform[0].connection_string
        }
      }
    }
  }

  tags = var.resource_tags
}

# ❌ WRONG: Conditional creation via true/false resource duplication
# resource "azurerm_container_app" "web_enabled" { ... }
# resource "azurerm_container_app" "web_disabled" { ... }

# ❌ WRONG: Using string conditions for defaults
# locals {
#   name = var.custom_name == "" ? "default-name" : var.custom_name
# }
# ✅ CORRECT:
# locals {
#   name = coalesce(var.custom_name, "default-name")
# }
```

### locals.tf — Organized for Reuse

```hcl
# ✅ CORRECT: Alphabetical ordering, typed precisely

locals {
  # Common naming patterns (used in multiple resources)
  app_name_short    = substr(var.environment, 0, 3)
  resource_prefix   = "${var.environment}-${local.app_name_short}"
  
  # Environment-specific defaults
  vm_count = var.environment == "prod" ? 3 : 1
  vm_sku   = var.environment == "prod" ? "Standard_D4s_v4" : "Standard_B2s"
  
  # Computed cost allocation
  common_tags = merge(
    var.resource_tags,
    {
      environment = var.environment
      deployed_by = "terraform"
      managed_by  = "platform-team"
    }
  )
}
```

## Bicep Modules (Microsoft Learn Standard)

### File Structure Template

```
webapp/infrastructure/
├── main.bicep                    # Orchestrates all modules
├── parameters.bicepparam         # Environment-specific values
├── containerRegistry.bicep       # ACR module
├── logAnalytics.bicep           # Log Analytics workspace module
├── containerAppsEnv.bicep       # Container Apps environment
├── containerApp.bicep           # Container App itself
├── keyVault.bicep               # Key Vault for secrets
├── appInsights.bicep            # Application Insights
└── README.md                    # Usage guide & examples
```

### Bicep Parameter Pattern (camelCasing)

```bicep
// ✅ CORRECT: Parameters at top with clear descriptions

param location string
param environment string  // 'dev', 'test', 'prod'
param containerImage string
param resourceNamePrefix string

@minLength(3)
@maxLength(24)
param storageAccountName string

@allowed([
  'Premium_LRS'
  'Standard_GRS'
  'Standard_LRS'
])
param storageReplication string = 'Standard_LRS'

param tags object = {
  environment: environment
  managedBy: 'bicep'
}

// ❌ WRONG: Variables for fixed values (use parameters instead)
// var defaultLocation = 'eastus'
```

### Bicep Outputs — Discrete Attributes Only

```bicep
// ✅ CORRECT: Output computed attributes (anti-corruption layer)

output registryLoginServer string = containerRegistry.outputs.loginServer
output registryId string = containerRegistry.outputs.registryId

output containerAppsEnvId string = containerAppsEnv.outputs.environmentId
output containerAppsEnvName string = containerAppsEnv.outputs.name

// Sensitive outputs (Bicep 0.35.1+)
@secure()
output keyVaultUri string = keyVault.outputs.vaultUri

// For array outputs
output containerAppUrls array = [
  for app in containerApps: {
    name: app.outputs.name
    url: app.outputs.ingressFqdn
  }
]

// ❌ WRONG: Export entire resource objects
// output containerRegistry object = containerRegistry  // Exposes all properties
```

### Bicep Module Pattern — Container Apps Example

```bicep
// containerRegistry.bicep
// Purpose: Azure Container Registry (ACR) for storing container images

param location string
param environment string
param resourceNamePrefix string

@allowed([
  'Basic'
  'Standard'
  'Premium'
])
param skuName string = 'Basic'

resource acr 'Microsoft.ContainerRegistry/registries@2023-07-01' = {
  name: '${resourceNamePrefix}acr${environment}'
  location: location
  sku: {
    name: skuName
  }
  properties: {
    adminUserEnabled: false
    publicNetworkAccess: 'Enabled'
    networkRuleBypassOptions: 'AzureServices'
  }
}

output loginServer string = acr.properties.loginServer
output registryId string = acr.id
output name string = acr.name
```

### Bicep main.bicep — Orchestration Pattern

```bicep
// main.bicep
// Purpose: Orchestrate all infrastructure modules for webapp

param location string
param environment string
param containerImage string

// Call modules in dependency order
module logAnalytics './logAnalytics.bicep' = {
  name: 'laDeployment'
  params: {
    location: location
    environment: environment
    resourceNamePrefix: 'hcw'
  }
}

module containerRegistry './containerRegistry.bicep' = {
  name: 'acrDeployment'
  params: {
    location: location
    environment: environment
    resourceNamePrefix: 'hcw'
  }
}

module keyVault './keyVault.bicep' = {
  name: 'kvDeployment'
  params: {
    location: location
    environment: environment
    resourceNamePrefix: 'hcw'
  }
}

module containerAppsEnv './containerAppsEnv.bicep' = {
  name: 'caeDeployment'
  params: {
    location: location
    environment: environment
    logAnalyticsWorkspaceId: logAnalytics.outputs.workspaceId  // Reference other module outputs
  }
}

module appInsights './appInsights.bicep' = {
  name: 'aiDeployment'
  params: {
    location: location
    environment: environment
    logAnalyticsWorkspaceId: logAnalytics.outputs.workspaceId
  }
}

module containerApp './containerApp.bicep' = {
  name: 'appDeployment'
  params: {
    location: location
    environment: environment
    containerImage: containerImage
    containerAppsEnvironmentId: containerAppsEnv.outputs.environmentId
    containerRegistryLoginServer: containerRegistry.outputs.loginServer
    keyVaultUri: keyVault.outputs.vaultUri
    appInsightsConnectionString: appInsights.outputs.connectionString
  }
}

// Output only the values needed downstream
output containerAppsUrl string = containerApp.outputs.ingressFqdn
output containerRegistryServer string = containerRegistry.outputs.loginServer
@secure()
output keyVaultUri string = keyVault.outputs.vaultUri
```

## Phase 1 Build: Node.js Backend Pattern

### Express Server Structure

```typescript
// backend/src/server.ts
import express, { Express, Request, Response } from 'express';
import * as dotenv from 'dotenv';

dotenv.config();

const app: Express = express();
const port = parseInt(process.env.PORT || '3001', 10);

// Middleware
app.use(express.json());

// Health check endpoint
app.get('/api/health', (req: Request, res: Response) => {
  res.status(200).json({
    status: 'healthy',
    timestamp: new Date().toISOString(),
    uptime: process.uptime(),
  });
});

// Bootstrap initialization endpoint
app.post('/api/bootstrap', (req: Request, res: Response) => {
  // Input validation
  const { orgName, azureSubscriptionId, region } = req.body;
  
  if (!orgName || !azureSubscriptionId || !region) {
    return res.status(400).json({
      error: 'Missing required fields: orgName, azureSubscriptionId, region',
    });
  }
  
  // TODO: Call bootstrap service
  const jobId = generateJobId();
  
  res.status(202).json({
    jobId,
    status: 'queued',
    message: 'Bootstrap initialization started',
  });
});

// Job status endpoint
app.get('/api/jobs/:id', (req: Request, res: Response) => {
  const { id } = req.params;
  
  // TODO: Query job store (in-memory for Phase 1)
  res.status(200).json({
    jobId: id,
    status: 'in_progress',
    progress: 45,
    message: 'Creating GitHub repository...',
  });
});

// GitHub OAuth callback
app.get('/api/auth/github/callback', (req: Request, res: Response) => {
  const { code } = req.query;
  
  if (!code) {
    return res.status(400).json({ error: 'Missing authorization code' });
  }
  
  // TODO: Exchange code for token via Octokit
  res.status(200).json({
    success: true,
    message: 'Authentication successful',
  });
});

// Error handling middleware
app.use((err: any, req: Request, res: Response, next: any) => {
  console.error(err);
  res.status(500).json({
    error: 'Internal server error',
    message: process.env.NODE_ENV === 'development' ? err.message : undefined,
  });
});

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});

function generateJobId(): string {
  return `job-${Date.now()}-${Math.random().toString(36).substr(2, 9)}`;
}
```

## Phase 1 Build: React Frontend Pattern

### Vite + React + TypeScript Structure

```typescript
// frontend/src/App.tsx
import React, { useState } from 'react';
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import LandingPage from './pages/LandingPage';
import Wizard from './pages/Wizard';
import StatusPage from './pages/StatusPage';

function App() {
  const [sessionToken, setSessionToken] = useState<string | null>(
    localStorage.getItem('session_token')
  );

  const handleGitHubAuth = (token: string) => {
    setSessionToken(token);
    localStorage.setItem('session_token', token);
  };

  const handleLogout = () => {
    setSessionToken(null);
    localStorage.removeItem('session_token');
  };

  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<LandingPage onAuth={handleGitHubAuth} />} />
        <Route
          path="/wizard"
          element={
            sessionToken ? (
              <Wizard token={sessionToken} />
            ) : (
              <LandingPage onAuth={handleGitHubAuth} />
            )
          }
        />
        <Route
          path="/status/:jobId"
          element={
            sessionToken ? (
              <StatusPage token={sessionToken} />
            ) : (
              <LandingPage onAuth={handleGitHubAuth} />
            )
          }
        />
      </Routes>
    </BrowserRouter>
  );
}

export default App;
```

## Phase 2 Build: Multi-Stage Dockerfile Pattern

```dockerfile
# Stage 1: Build frontend
FROM node:20-alpine AS frontend-builder
WORKDIR /app/frontend
COPY webapp/frontend/package*.json ./
RUN npm ci
COPY webapp/frontend/ ./
RUN npm run build

# Stage 2: Build backend
FROM node:20-alpine AS backend-builder
WORKDIR /app/backend
COPY webapp/backend/package*.json ./
RUN npm ci
COPY webapp/backend/tsconfig.json ./
COPY webapp/backend/src ./src
RUN npm run build

# Stage 3: Runtime
FROM node:20-alpine
RUN addgroup -g 1001 -S nodejs && adduser -S nodejs -u 1001

WORKDIR /app

# Copy built backend
COPY --from=backend-builder --chown=nodejs:nodejs /app/backend/dist ./dist
COPY --from=backend-builder --chown=nodejs:nodejs /app/backend/package*.json ./

# Copy built frontend static assets
COPY --from=frontend-builder --chown=nodejs:nodejs /app/frontend/dist ./public

# Install only production dependencies
RUN npm ci --only=production

USER nodejs
EXPOSE 8080

HEALTHCHECK --interval=30s --timeout=3s --start-period=40s --retries=3 \
  CMD node -e "require('http').get('http://localhost:8080/api/health', (r) => {if (r.statusCode !== 200) throw new Error(r.statusCode)})"

CMD ["node", "dist/server.js"]
```

## GitHub Actions Workflow Pattern (Phase 2)

**File**: `.github/workflows/webapp-deploy.yml`

```yaml
name: Deploy Web App to Azure

on:
  push:
    branches: [main]
    paths:
      - 'webapp/**'
      - '.github/workflows/webapp-deploy.yml'
  workflow_dispatch:

env:
  REGISTRY: ${{ secrets.AZURE_CONTAINER_REGISTRY_LOGIN_SERVER }}
  IMAGE_NAME: webapp

jobs:
  build-and-push:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      id-token: write

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v2

      - name: Azure Login
        uses: azure/login@v1
        with:
          client-id: ${{ secrets.AZURE_CLIENT_ID }}
          tenant-id: ${{ secrets.AZURE_TENANT_ID }}
          subscription-id: ${{ secrets.AZURE_SUBSCRIPTION_ID }}

      - name: Login to ACR
        run: |
          az acr login --name ${{ secrets.AZURE_CONTAINER_REGISTRY_NAME }}

      - name: Build and push image
        uses: docker/build-push-action@v4
        with:
          context: .
          platforms: linux/amd64,linux/arm64
          push: true
          tags: |
            ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:${{ github.sha }}
            ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:latest
          cache-from: type=gha
          cache-to: type=gha,mode=max

      - name: Deploy to Container Apps
        run: |
          az containerapp update \
            --resource-group ${{ secrets.AZURE_RESOURCE_GROUP }} \
            --name hcw-webapp \
            --image ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:${{ github.sha }}

      - name: Smoke test
        run: |
          sleep 30
          curl -f https://${{ secrets.AZURE_CONTAINER_APP_FQDN }}/api/health || exit 1
```

## Summary: Before You Build

1. ✅ **Complete Phase 0** — Audit existing repo against standards
2. ✅ **Review Terraform modules** — Check each against AVM checklist
3. ✅ **Plan Bicep structure** — Design modules for webapp (Phase 2)
4. ✅ **Follow these templates** — Use exact patterns for Phase 1+

**No build scripts execute until Phase 0 audit complete.**
